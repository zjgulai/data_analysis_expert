## 一、知识库探测阶段

### 1.1 探测目标

### 1.2 探测策略

**分层探测法**:
```
Phase 1: 主题探测 (广泛)
```

### 3.2 创建的Skills详情

#### Skill 1: channel-effect-analysis

**知识来源**: Biji知识库
**内容长度**: 902字
**核心内容**:
- 渠道类型与分类
- 关键指标: ROI, CAC, 留存率
- 归因模型: 首次触点、末次触点、线性归因
- 多渠道整合分析方法

#### Skill 2: traffic-source-analysis

**知识来源**: Biji知识库
**内容长度**: 874字
**核心内容**:
- 流量来源分类: 直接访问、搜索引擎、社交媒体等
- 流量质量评估三级体系
- 转化漏斗分析方法

#### Skill 3: ad-performance-analysis

**知识来源**: Biji知识库
**内容长度**: 1053字 (最丰富)
**核心内容**:
- 广告效果指标: CTR, CPC, CVR, ROAS
- 广告投放优化策略
- 动态ROI模型
- A/B测试方法

#### Skill 4: social-media-analysis

**知识来源**: Biji知识库
**内容长度**: 811字
**核心内容**:
- 社媒效果指标: 粉丝增长率、互动率
- 内容分析方法
- 用户画像构建

#### Skill 5: supplier-performance

**知识来源**: Biji知识库
**内容长度**: 718字
**核心内容**:
- 供应商评估指标: 质量、交付、成本、服务
- 供应商五分法: 战略、优选、资格未定、被动淘汰、主动淘汰
- 绩效管理策略

#### Skill 6: risk-warning-system

**知识来源**: Biji知识库
**内容长度**: 559字
**核心内容**:
- 风险指标体系: 1+6+4+3结构
- 预警规则设计: 乐观/保守/悲观
- 风险应对策略: 规避、承受、利用、减小、分担

#### Skill 7: complex-contribution

**知识来源**: 数学推导 + 业务需求
**创新点**: 偏微分+全微分方法
**核心内容**:
- 任意四则运算关系的贡献度计算
- 静态贡献度（偏导数）
- 波动贡献度（全微分）
- 交互效应识别

## 四、知识萃取方法论总结

### 4.1 萃取流程

```
1. 知识收集
   ├── 通过API查询获取相关内容
   ├── 记录来源和引用
   └── 保存原始数据

2. 结构化整理
   ├── 提取核心概念
   ├── 识别方法论步骤
   └── 整理公式和模型

3. Skill设计
   ├── 确定Skill功能边界
   ├── 设计输入输出规范
   └── 编写示例场景

4. 验证与完善
   ├── 检查与现有Skills的边界
   ├── 确保不重复
   └── 补充行业特性
```

### 4.2 Skill模板规范

```markdown
---
name: cbec-[skill-name]
description: |
  [功能描述]

  触发条件:
  1. [条件1]
  2. [条件2]

  输出内容:
  - [输出1]
  - [输出2]
---

# [Skill名称]

## 概述
## 核心功能
## 使用方法
## 核心公式/方法
## 示例
## 注意事项
## 参考资料
```

### 4.3 质量检查清单

- [ ] 功能定义清晰，边界明确
- [ ] 与现有Skills不重复
- [ ] 包含完整的输入输出规范
- [ ] 至少包含2-3个示例场景
- [ ] 包含核心公式或方法论
- [ ] 包含注意事项和限制说明
- [ ] 标注知识来源(Biji知识库)
