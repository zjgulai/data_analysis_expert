# 麦肯锡多维对比分析图表 Skill

## 概述

多维对比分析图表是麦肯锡深度分析的核心工具，能够在单一视图中展示复杂的数据关系，帮助决策者快速理解业务全貌。

## 核心理念

```
单一复合图表 > 多个子图表

优势:
• 信息密度更高
• 关联关系更清晰
• 结论更直观
• 决策更高效
```

## 八大多维分析图表类型

### 1. 增长驱动分解图 (Growth Driver Decomposition)

**图表类型**: 堆叠柱状图 / 瀑布图

**表达结论**:
- 销额增长由哪些因素驱动
- 各因素贡献度大小
- 正负因素对比

**适用场景**:
```
✓ 销额增长归因分析
✓ 指标变化因素分解
✓ 财务分析（收入/成本构成）
✓ 用户增长驱动分析

✗ 不适合: 单一因素分析
```

**结构示例**:
```
增长驱动分解（销额变化）

+200万 ┤              ████
       │         ████ ████ 渗透率 +15%
+100万 ┤    ████ ████ ████
       │    ████ ████ ████ 购买频次 +8%
   0   ┤────████ ████ ████────────────
       │    ▓▓▓▓ ▓▓▓▓ ▓▓▓▓
-100万 ┤    ▓▓▓▓ ▓▓▓▓ ▓▓▓▓ 均价 -5%
       │    ▓▓▓▓ ▓▓▓▓ ▓▓▓▓
       └────────────────────────
          Q1    Q2    Q3

  █ 正向因素  ▓ 负向因素
```

**母婴电商案例**:
```markdown
# 销额增长主要由购买频次提升(+15%)和客单价上涨(+8%)驱动

## 通过图表可知
• 购买频次贡献+15%，是最大增长驱动因素
• 客单价上涨贡献+8%，高端化趋势明显
• 渗透率下降-5%，新客获取不足
• 建议加强老客运营，同时优化获客策略

## 数据结构
├── 渗透率变化: -5%
├── 购买频次变化: +15% (主力驱动)
├── 客单价变化: +8%
└── 总销额增长: +18%
```

### 2. 竞争转换分析图 (Competitive Switching Analysis)

**图表类型**: 双向条形图 / 桑基图

**表达结论**:
- 消费者在品牌间的流动方向
- 流失/获得规模对比
- 主要竞争威胁来源

**适用场景**:
```
✓ 品牌竞争分析
✓ 用户流失/获得分析
✓ 市场份额变化原因
✓ 竞品对标分析

✗ 不适合: 单一品牌分析
```

**结构示例**:
```
竞争转换分析（消费者流向）

获得 ↑
     │     ████████████████████ ← 从简爱获得
     │     ████████████████     ← 从君乐宝获得
     │     ████████             ← 从光明获得
─────┼──────────────────────────────────────
     │               ▓▓▓▓▓▓▓▓ → 流失给简爱
     │               ▓▓▓▓▓▓▓▓▓▓ → 流失给君乐宝
     │               ▓▓▓▓▓▓ → 流失给光明
     ↓ 流失

  █ 获得  ▓ 流失  净值 = 获得 - 流失
```

**母婴电商案例**:
```markdown
# 卡士主要流失给简爱(+200万)和君乐宝(+150万)，需重点关注高端竞品

## 通过图表可知
• 净流失给简爱200万，是最大竞争威胁
• 从常温酸奶获得用户100万，说明低温酸奶有替代优势
• 净流失给君乐宝150万，主要流向其无糖产品线
• 建议：加强无糖产品线布局，抵御简爱抢夺

## 关键词
Competitive Switching | User Flow | Brand Migration
```

```
✓ 竞品价格对标

✗ 不适合: 单一价格点分析
```

### 4. 消费者特征矩阵图 (Consumer Profile Matrix)

**图表类型**: 散点图 + 100%堆叠柱状图组合

**表达结论**:
- 目标群体特征分布
- 不同群体行为差异
- 人群定位精准度

**适用场景**:
```
✓ 流失/新进买者画像
✓ 目标人群分析
✓ 用户分群对比
✓ 人群偏好差异分析

✗ 不适合: 单一维度分析
```

**结构示例**:
```
流失买者特征分析

散点图（重要性 vs 表现）
重要性
  ↑
  │         ● 店内陈列
  │    ● 广告提醒
  │              ● 产品成分
  │         ● 价格促销
  └────────────────────→ 表现
     差              好

100%堆叠图（渠道分布）
     │ ████████████████████████████
     │ ▓▓▓▓▓▓▓▓████████████████████
     │ ░░░░░░░░▓▓▓▓▓▓▓▓████████████
     │ ████████░░░░░░░░▓▓▓▓▓▓▓▓████
     └────────────────────────────
        流失买者  留存买者  新进买者

  █大卖场  ▓O2O  ░线上  █便利店
```

**母婴电商案例**:
```markdown
# 流失买者更关注产品成分(8.5分)和广告提醒(7.8分)，但卡士表现不足

## 通过图表可知
• 流失买者对产品成分重要性评分8.5，但卡士仅6.2分
• 广告提醒重要性7.8分，卡士表现5.5分，差距明显
• 流失买者更多来自大卖场渠道(45%)
• 建议：加强产品成分沟通，增加广告投放频次

## 关键词
Churn Analysis | Consumer Profile | Attribute Importance
```

### 7. 瀑布图 (Waterfall Chart)

**图表类型**: 瀑布图 / 阶梯图

**表达结论**:
- 从期初到期末的变化过程
- 各因素累积影响
- 最终结果构成

**适用场景**:
```
✓ 财务分析（收入/利润变化）
✓ 销售变化归因
✓ 成本构成分析
✓ 预算vs实际对比

✗ 不适合: 单一因素分析
```

**结构示例**:
```
利润变化瀑布图

         ┌─────┐
         │ +50 │ 均价提升
         └──┬──┘
      ┌─────┴─────┐
      │    +30    │ 销量增长
      └─────┬─────┘
   ┌────────┴────────┐
   │       -20       │ 成本上升
   └────────┬────────┘
┌───────────┴───────────┐
│        期初利润        │
│          100          │
└───────────────────────┘

最终利润 = 100 + 50 + 30 - 20 = 160
```

**母婴电商案例**:
```markdown

• 期初毛利: $50万
• 均价提升: +$10万 (最大贡献)
• 销量增长: +$8万
• 成本上升: -$3万 (抵消因素)
• 期末毛利: $65万

## 关键词
Waterfall | Profit Analysis | Gross Margin
```

## 复合图表 vs 多子图表选择指南

### 选择决策框架

```
                数据维度
                   │
         ┌────────┴────────┐
         │                 │
     2-3个维度         4+个维度
         │                 │
    ┌────┴────┐       ┌────┴────┐
    │         │       │         │
 单一复合图  多子图   单一复合图  多子图
    │         │       │         │
    ▼         ▼       ▼         ▼
  优先      可选     谨慎      优先
```

### 具体选择建议

| 场景 | 推荐方案 | 原因 |
|-----|---------|------|
| 增长驱动分析 | 单一堆叠图 | 因素间关系清晰 |
| 竞争转换分析 | 单一双向条形图 | 流向关系直观 |
| 多属性对比(>6个) | 多子图组合 | 避免信息过载 |
| 时间趋势+增长率 | 单一复合图 | 规模与效率关联 |
| 多渠道多指标 | 多子图组合 | 维度太多 |
| 价格带+趋势 | 单一复合图 | 结构变化清晰 |

### 设计原则

```
1. 信息密度原则
   • 单图表承载 ≤ 5个维度
   • 超过则拆分为多子图

2. 关联性原则
   • 维度间有逻辑关联 → 复合图
   • 维度独立 → 多子图

3. 可读性原则
   • 关键结论一目了然
   • 辅助信息不干扰主信息

4. 一致性原则
   • 多子图保持统一风格
   • 颜色编码一致
```

## 技术实现

### Python (Matplotlib) - 瀑布图

```python
import matplotlib.pyplot as plt
import numpy as np

# 数据
categories = ['期初', '均价提升', '销量增长', '成本上升', '期末']
values = [100, 50, 30, -20, 160]

# 计算位置
cumulative = [0]
for v in values[1:-1]:
    cumulative.append(cumulative[-1] + v)

# 绘制
fig, ax = plt.subplots(figsize=(10, 6))

colors = ['#051C2C', '#3A5F8A', '#3A5F8A', '#FF6B35', '#051C2C']

for i, (cat, val, cum) in enumerate(zip(categories[:-1], values[:-1], cumulative)):
    if val >= 0:
        ax.bar(cat, val, bottom=cum, color=colors[i])
    else:
        ax.bar(cat, val, bottom=cum + val, color=colors[i])

# 期末
ax.bar(categories[-1], values[-1], color=colors[-1])

plt.ylabel('毛利(万元)')
plt.title('利润变化瀑布图')
plt.tight_layout()
plt.show()
```
